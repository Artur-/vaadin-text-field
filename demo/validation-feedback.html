Validation Feedback

Validation error when blurred

This required field is validated after the user blurs the field. Feedback is shown only in the case of a validation error.

Validation success and error feedback when blurred

The username field should be validated (usually on the server) to guarantee uniqueness. Feedback is shown for both validation errors and successful validation.

Immediate success feedback and late failure feedback

A valid email address is indicated immediately and an invalid email address is indicated on blur.

Immediate feedback for both valid and invalid values

A numbers-only field should be validated on each input event, to prompt the user to correct the input immediately.

Lazy feedback

The validation feedback can also be triggered lazily only when explicitly requested, for example when submitting a form.
